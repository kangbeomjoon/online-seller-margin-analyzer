# 샘플 데이터 설계 문서

작성일: 2026-05-01
프로젝트: 온라인 셀러 정산·마진 자동 분석기
기준 문서: `docs/portfolio-planning.md`

## 1. 목적

이 문서는 첫 포트폴리오 데모에서 사용할 샘플 CSV 데이터의 구조와 계산 기준을 정의한다.

1차 데모는 심플하게 스마트스토어와 쿠팡 2개 플랫폼만 포함한다. 자사몰은 향후 확장 가능한 플랫폼 예시로 남기고, 첫 구현 범위에는 넣지 않는다.

샘플 데이터는 단순 더미가 아니라 앱의 핵심 기능을 검증하는 기준 데이터로 사용한다. 따라서 정상 주문뿐 아니라 저마진, 적자, 원가 미등록, 배송비 누락 같은 운영상 문제 케이스를 일부러 포함한다.

## 2. 1차 샘플 파일 목록

| 파일 | 역할 | 위치 |
|---|---|---|
| 스마트스토어 주문 CSV | 스마트스토어 원본 주문 파일 예시 | `sample-data/smartstore-orders.csv` |
| 쿠팡 주문 CSV | 쿠팡 원본 주문 파일 예시 | `sample-data/coupang-orders.csv` |
| 상품 원가표 | 상품별 매입원가와 부대비용 기준 | `sample-data/product-costs.csv` |
| 수수료 설정표 | 플랫폼별 수수료율과 고정비 기준 | `sample-data/fee-rules.csv` |

## 3. 데이터 규모

1차 데모 데이터는 다음 규모로 만든다.

| 항목 | 기준 |
|---|---:|
| 기간 | 2026-01 ~ 2026-03, 3개월 |
| 플랫폼 | 스마트스토어, 쿠팡 |
| 상품 수 | 10개 |
| 옵션 수 | 14~18개 |
| 주문 수 | 80~120건 |
| 원가 미등록 케이스 | 2~4건 |
| 배송비 누락 케이스 | 2~4건 |
| 적자 상품 케이스 | 2~3개 상품 |
| 저마진 상품 케이스 | 2~3개 상품 |

이 규모는 차트와 테이블이 비어 보이지 않으면서도, 사람이 CSV를 열어 계산 흐름을 검토할 수 있는 수준이다.

## 4. 공통 원칙

### 4.1 통화와 숫자

- 모든 금액은 원화 KRW 기준 정수로 저장한다.
- CSV에는 쉼표 없는 숫자를 사용한다. 예: `12900`
- 앱 화면에서만 천 단위 쉼표와 `원` 단위를 붙인다.

### 4.2 날짜

- 원본 CSV 날짜는 플랫폼별 컬럼명을 유지한다.
- 값 형식은 모두 `YYYY-MM-DD`로 통일한다.
- 내부 표준 데이터에서는 `orderDate`와 `orderMonth`를 사용한다.

### 4.3 주문 상태

1차 데모 계산에는 매출로 인정할 수 있는 유효 주문만 포함한다.

| 원본 상태 | 내부 상태 | 매출/마진 계산 포함 여부 |
|---|---|---|
| 배송완료 | completed | 포함 |
| 구매확정 | completed | 포함 |
| 결제완료 | paid | 포함 |
| 배송중 | shipping | 포함 |
| 취소 | canceled | 제외 |
| 반품 | returned | 제외 |

취소와 반품 주문은 샘플 CSV에 소량 포함하되 KPI와 마진 계산에서는 제외한다. 대시보드에는 제외 건수를 검증 정보로 보여줄 수 있다.

### 4.4 상품 매칭

1차 데모에서는 원본 주문 CSV에 `상품코드`가 없다고 가정한다. 앱은 `상품명 + 옵션명` 조합으로 상품 원가표와 매칭한다.

내부 계산 시에는 상품 원가표의 `상품코드`를 표준 주문 데이터에 붙인다.

매칭 실패 조건:

- 주문 CSV의 `상품명 + 옵션명` 조합이 원가표에 없는 경우
- 상품명은 같지만 옵션명이 비어 있거나 다르게 적힌 경우

매칭 실패 주문은 계산에서 제외하지 않는다. 대신 원가를 `0`으로 채우지 않고 `profitStatus = missing_cost`로 표시하며, 경고 목록에 노출한다.

## 7. 상품 원가표

파일: `sample-data/product-costs.csv`

### 7.1 컬럼

| 컬럼명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 상품코드 | string | 예 | 내부 상품 식별자 |
| 상품명 | string | 예 | 주문 CSV와 매칭할 상품명 |
| 옵션명 | string | 아니오 | 주문 CSV와 매칭할 옵션명 |
| 매입원가 | number | 예 | 상품 1개당 매입원가 |
| 기본배송비 | number | 예 | 주문 1건당 판매자가 부담하는 기본 배송비 |
| 포장비 | number | 예 | 상품 1개당 포장비 |
| 기타비용 | number | 예 | 상품 1개당 기타 비용 |

### 7.2 예시 행

```csv
상품코드,상품명,옵션명,매입원가,기본배송비,포장비,기타비용
P-001,프리미엄 무선 충전기,화이트,9200,3000,500,300
P-002,휴대용 미니 가습기,핑크,12500,3000,600,400
```

### 7.3 비용 적용 기준

- `매입원가`, `포장비`, `기타비용`은 수량을 곱한다.
- `기본배송비`는 주문 1건당 1회 적용한다.
- 1차 샘플의 모든 상품은 실물 배송 상품으로 본다.
- 따라서 `기본배송비`가 `0`이거나 비어 있는 상품은 배송비 기준 누락 의심 대상으로 표시한다.
- 합배송이나 묶음배송 최적화는 1차 데모에서 제외한다.

## 8. 수수료 설정표

파일: `sample-data/fee-rules.csv`

### 8.1 컬럼

| 컬럼명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| 플랫폼 | string | 예 | `smartstore` 또는 `coupang` |
| 기본수수료율 | number | 예 | 판매금액 기준 비율, 예: 0.035 |
| 결제수수료율 | number | 예 | 결제금액 기준 비율, 예: 0.018 |
| 주문당고정비 | number | 예 | 주문 1건당 고정 비용 |

### 8.2 예시 행

```csv
플랫폼,기본수수료율,결제수수료율,주문당고정비
smartstore,0.035,0.018,0
coupang,0.085,0.022,150
```

### 8.3 수수료 계산 기준

- 기본 수수료 = `grossSales * 기본수수료율`
- 결제 수수료 = `paidAmount * 결제수수료율`
- 고정비 = `주문당고정비`
- 수수료는 원 단위 반올림한다.

## 10. 계산 공식

### 10.1 매출 기준

1차 데모의 매출은 `grossSales - discountAmount`로 계산한다. 고객 청구 배송비는 매출 KPI에는 포함하지 않고, 배송비 수지 분석은 향후 확장으로 둔다.

```text
netSales = grossSales - discountAmount
```

### 10.2 비용 기준

```text
productCost = unitCost * quantity
packagingTotal = packagingCost * quantity
otherTotal = otherCost * quantity
salesFee = round(grossSales * baseFeeRate)
paymentFee = round(paidAmount * paymentFeeRate)
fixedFee = fixedFeePerOrder
totalCost = productCost + shippingCost + packagingTotal + otherTotal + salesFee + paymentFee + fixedFee
```

### 10.3 순이익과 마진율

```text
netProfit = netSales - totalCost
marginRate = netProfit / netSales
```

`netSales`가 0 이하인 주문은 마진율을 계산하지 않고 검증 경고에 표시한다.

## 11. 상태 분류 기준

| 상태 | 조건 | 화면 표시 |
|---|---|---|
| `healthy` | `marginRate >= 0.20` | 정상 |
| `low_margin` | `0 <= marginRate < 0.20` | 저마진 |
| `loss` | `netProfit < 0` | 적자 |
| `missing_cost` | 원가표 매칭 실패 | 원가 미등록 |
| `excluded` | 취소 또는 반품 | 계산 제외 |

저마진 기준은 20%로 고정한다. 첫 포트폴리오에서는 사용자가 임계값을 바꾸는 기능을 넣지 않는다.

## 12. 의도적으로 포함할 데모 시나리오

| 시나리오 | 목적 |
|---|---|
| 스마트스토어에서는 흑자인데 쿠팡에서는 저마진인 동일 상품 | 플랫폼별 수수료 차이를 보여준다 |
| 많이 팔리지만 순이익이 낮은 상품 | 매출만 보면 놓치는 문제를 보여준다 |
| 판매량은 적지만 고마진인 상품 | 상품별 전략 판단 가능성을 보여준다 |
| 원가표에 없는 상품 | 수작업 누락 검증의 가치를 보여준다 |
| 상품 원가표의 기본배송비가 0으로 등록된 배송 상품 | 배송비 누락 점검의 가치를 보여준다 |
| 취소/반품 주문 | 계산 제외 기준을 보여준다 |
| 3개월간 순이익이 개선되는 흐름 | 월별 대시보드 차트를 풍성하게 만든다 |

## 13. 검증 경고 데이터

대시보드의 경고/검증 화면은 다음 목록을 생성한다.

| 경고 | 조건 |
|---|---|
| 원가 미등록 상품 | 원가표 매칭 실패 |
| 배송비 누락 의심 | 배송 상품인데 상품 원가표의 기본배송비가 0이거나 비어 있음 |
| 수수료 설정 누락 플랫폼 | 주문의 플랫폼이 수수료 설정표에 없음 |
| 적자 상품 TOP 10 | 상품별 집계 순이익이 0 미만 |
| 저마진 상품 TOP 10 | 상품별 집계 마진율이 0 이상 20% 미만 |
| 계산 제외 주문 | 주문상태가 취소 또는 반품 |

## 14. 1차 제외 범위

다음 항목은 데이터 설계에 반영하지 않는다.

- 실제 스마트스토어/쿠팡 원본 CSV 완전 호환
- 부가세, 매출세액, 종합소득세 등 세무 신고 수준 계산
- 쿠팡 정산 보류, 반품 배송비, 부분 취소 정산
- 합배송, 묶음배송, 지역별 추가 배송비
- 재고 수량과 매입 시점별 원가 변동
- 옵션명이 불규칙한 상품의 자동 정규화
- 자사몰 CSV

## 15. 완료 기준

샘플 데이터 작성이 완료되면 다음 조건을 만족해야 한다.

- 4개 CSV 파일이 모두 존재한다.
- 스마트스토어와 쿠팡 주문이 모두 표준 주문 데이터로 변환된다.
- 3개월 월별 차트가 의미 있게 표시된다.
- 플랫폼별 차트가 의미 있게 표시된다.
- 상품별 마진 테이블에 정상, 저마진, 적자 상태가 모두 나온다.
- 경고 화면에 원가 미등록, 배송비 누락, 계산 제외 주문이 나온다.
- 같은 샘플 데이터로 엑셀 다운로드 결과를 만들 수 있다.
